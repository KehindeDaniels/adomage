3. How it‚Äôs used in updates

Whenever we make a destructive change (add, update, delete, reorder), we do:

  [...state.history.past, state.history.present],
  state.history.limit

  text,
  history: {
    past,
    present: snapshot(state.project, text),

Step by step:

  1. Take the current present (the editor right now).
  2. Push it into the past array (so you can undo back to it).
  3. Trim if needed (don‚Äôt let it grow beyond limit).
  4. Create a new present snapshot for the updated state.
  5. Clear future (because once you make a new change, redo stack is invalid).

4. The 3 parts of history

  • past: list of previous snapshots (old states)
  • present: the current snapshot (right now)
  • future: states you undid and could redo

It looks like this:

past:    [ S1, S2, S3 ]
present: S4
future:  [ S5, S6 ]
  • Undo = move S4 into future, take last from past into present.
  • Redo = move from future back into present.

5. Example workflow

  1. Start with blank image ‚Üí present = empty

  2. Add a text layer:

    • old present goes into past
    • new state becomes new present

  3. Update text content:

    • old present goes into past
    • new updated state becomes present

  4. Press Undo:

    • move present into future
    • pop last from past ‚Üí becomes present again

  5. Press Redo:

    • take first from future ‚Üí move it back into present
